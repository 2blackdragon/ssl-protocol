# Программная реализация протокола SSL

Цель проекта — наглядно показать основные этапы работы протокола:

- генерация и проверка самоподписанных сертификатов
- обмен сообщениями рукопожатия (handshake)
- аутентификация сервера (обязательная) и клиента (опциональная)
- обмен сессионными ключами через RSA
- симметричное шифрование трафика (AES-256-CBC)
- проверка целостности сообщений (HMAC-SHA256)
- защищённый обмен текстовыми сообщениями

## Основные возможности

- Клиент-серверное приложение с графическим интерфейсом (Tkinter)
- Поддержка опциональной аутентификации клиента
- Полное логирование всех этапов рукопожатия и обмена данными
- Генерация и загрузка самоподписанных сертификатов
- Шифрование и расшифровка сообщений в реальном времени
- Проверка целостности каждого пакета через HMAC
